MeetUp

Overview

The goal of this project is to build a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. The application uses the Google Calendar API to fetch upcoming events.

Purpose & Context

MeetUp is a serverless, progressive web application. I used Jest test driven development while creating this application. I also wanted to learn about OAuth2 and Google Calender API. Providing limited information to the customers even when they are not connected to the internet.

Objective

The aim of the project was to learn about PWA. Using test driven development writing test cases and user scenarios. Learning about user authentication with google OAuth2. I also wanted to learn about showing statistics in an application so I used recharts library.

Approach

SERVER-SIDE

For Back-End I serverless functions (AWS lambda is preferred) for the authorization server instead of using a traditional server. I also created a new project on Google API console to implement calendar API and OAuth2. I created credentials, scopes and added origin and URI to apps domain.

CLIENT-SIDE

After completing the Back-End part, I chose React Native as its best suited for the view side of the mvc approach and its virtual DOM ensures faster rendering of views. The component-based nature of React allows me to increase the scope of my web application with little to no performance issues or concerns about entropy. The interface lets users requests to, and receive responses from, the server-side.

Challenges

One of the challenge I faced was using serverless functions to adhere to Google OAuth2 authentication flow. It was difficult to understand the process of using a serverless functions (AWS lambda is preferred) for the authorization server instead of using a traditional server.

I was also struggling with understanding the concept of providing data offline or in slow network conditions with the help of a service worker.

Duration

It took me about two months to develop this application as I was using several new technologies on which I had never worked before. It was hard to learn the concepts and get a hold of it.

Retrospective

Looking back at the whole process the most complicated part was to setup the Back-End process.

Creating serverless functions to adhere to Google OAuth2 authentication was really time consuming and challenging. Having no previous knowledge about how things worked it was a big learning curve for me to implement new technologies such as OAuth2, AWS Lambda and writing test scenarios.The most fascinating part was to provide data offline or with limited internet service.

This experience will clearly help me to develop future projects. I will be able to follow the optimal design flow of which technologies to choose depending on the complexity of the project as well as additional libraries that can be integrated.
