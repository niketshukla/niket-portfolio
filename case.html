MyFlixStudio

Overview

This is a web application, created with React Js, will provide users with access to information about different movies, directors, and genres. Users will be able to sign up or login, update their personal information, and create a list of their favourite movies. It's based on its existing server-side code (REST API and database).

Purpose & Context

MyflixStudio is a personal full-stack web application. Where I got to learn and practice my Front-End as well as Back-End skills. From creating an API and database and hosting them on online services to creating front- end using React Js. This whole process made me more clear about the whole working of a web application from a Back-End as well as Front-End perspective.

Objective

The aim of the project was to have an ambitious full-stack project I can add to my professional portfolio. The problem I wanted to solve is to build the complete server-side and client-side for the application from scratch.

Approach

SERVER-SIDE

For Back-End I created RESTful API using Node Js and Express Js. As they are most compatible with React Js which I used as a Front-End technology. To host the API, I used Heroku which is a cloud platform that supports Node Js. After that I created database using MongoDB and connected API to database by hosting it on MongoDB cloud.

To test the API, I used Postman. I also included user authentication and authorization code in the form of basic HTTP authentication and JWT authentication.

CLIENT-SIDE

After completing the Back-End part, I chose React Js as its best suited for the view side of the mvc approach and its virtual DOM ensures faster rendering of views. The component-based nature of React allows me to increase the scope of my web application with little to no performance issues or concerns about entropy. The interface lets users requests to, and receive responses from, the server-side. It is a single-page, responsive application.

Challenges

One of the challenge I faced was creating a database on terminal with mongoose commands. It was difficult as I had to use long scripts and API references in them to create each record in the database collection. After creating the database I was facing problem connecting it to API as it was returning no record while accessing. So I had to verify the correct database name I used to create all the collections and records. Turns out that despite create a new database for my records it stored all the data in the test database.

I was also struggling with database key names being referenced on the Front-End part using React Js. Every key value is case sensitive so when mentioning them on few occasions the keys mentioned were not case sensitive and so it didn’t give me the desired result. After discussing this with my mentor I was able to solve this issue.

Duration

For the server side development I had to deal with fairly new technologies. So it took me more time then the estimated time given in the course.For the client side I had some prior experience with React Js so I was able to pick up the pace and complete it within stipulated time period.

Retrospective

Looking back at the whole process the most complicated part was to setup the Back-End process.

Creating API and database for the application was really time consuming and challenging. Having no previous knowledge about how things worked it was a big learning curve for me to implement new technologies such as Node Js, Express Js and Mongo DB. The most fascinating part was that for the first time I was actually connecting my Front-End to the Back-End data. To know how the information flows when an application makes an API request.

This experience will clearly help me to develop future projects. I will be able to follow the optimal design flow of which technologies to choose depending on the complexity of the project as well as additional libraries that can be integrated.
