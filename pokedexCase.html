Pokedex

Overview

JavaScript application to list different pokemons extracted with the help of external api.

Purpose & Context

In Pokedex i learned about how to use API's and show data. How to work with the documentation of various API's and what to look for according to our requirements.

Objective

The aim of the project was to relearn the basics of JavaScript. The problem I wanted to solve is to build an application using an API.

Approach

Client-Side

I started developing the structure for the project using HTML and CSS. Then I started implementing the functionality I wanted using JavaScript. After implementing the basic functionalities such as click on a button and showing a popup I a turned my attention to API's.

Challenges

For the most part I didn't face any challenges. I wanted to extract information from API using the best method possible. That is where I was confused which method to approach. I used Promises in JavaScript to get data.

Duration

It took me around 15 days to complete this project. The complexity of the project was low and the area of concerns were less so it didn't take much time to implement the functionalities.

Retrospective

It was a really nice experience to brush up my skills and learn about Immediately Invoked Function Expression, API's and the best suitable method to get data using Promises. I would like to use AXIOS in my next project to see what difference does it make in terms of reusable code and performance.
